## Why MIT?

We picked MIT for v0.0.1 because:

- It's the most permissive widely-recognised OSS license.
- It removes friction for anyone evaluating, embedding, or shipping internal forks.
- It's compatible with the existing TS ecosystem (most libs we depend on are MIT or Apache).

If we ever want to commercialize a hosted version and protect against AWS-style cloning, the path is to re-license **future versions** under [BSL](https://mariadb.com/bsl11/) or [FSL](https://fsl.software). Existing MIT versions stay MIT — they don't retroactively change.
