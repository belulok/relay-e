Relay-E is the **layer between an LLM and your customers' data**. It pulls relevant context before each turn, routes between "answer directly" and "run a multi-step agent," and keeps the call graph cheap, observable, and durable.

## What's in the box

| Layer | What it does |
|---|---|
| **Provider abstraction** | Anthropic / OpenAI / OpenRouter / Ollama via the Vercel AI SDK |
| **Skills + Tools** | Composable units of behaviour with Zod-validated input |
| **Context Resolver** | Parallel fetch from multiple sources, trimmed to a token budget |
| **Agent loop** | Tool-calling cycle with usage accounting and a typed event stream |
| **OpenAPI 3.1** | Auto-generated from Zod — Postman imports in one click |
| **Multi-tenant** | `tenant_id` flows through every layer; Postgres schema is RLS-ready |
| **Durable runs** | BullMQ + Redis for queued / long-running work and HITL approvals |

Built in TypeScript, runs on Node 22, MIT licensed.
